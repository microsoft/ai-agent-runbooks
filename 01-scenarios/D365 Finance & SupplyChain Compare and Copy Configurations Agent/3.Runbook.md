# Step-by-Step Guide — Dynamics 365 ERP Configurations Diagnostics Agent

> **Scenario**: ERP Configuration Diagnostics & Governance
> **Platform**: Microsoft Copilot Studio, Dynamics 365 F&O (v10.0.47), D365 ERP MCP Server (GA)
> **Target Readers**: Delivery engineer, Functional consultant, Solution architect
> **Last Updated**: April 28, 2026

---

## Overview

D365 F&O implementation teams are responsible for configuring hundreds of module parameters
across multiple legal entities. Inconsistent or missing configurations are among the most
common root causes of defects surfaced during UAT and post go-live. This agent makes
parameter governance easier by:

- Diagnosing what is configured versus what is missing per module, per company
- Comparing parameters across multiple companies side-by-side
- Assigning a risk level (High / Medium / Low) with a clear rationale
- Producing a prioritized 30-60-90-day remediation plan
- Updating specific parameters on approval, with confirmation and follow-up

In this step-by-step guide, you will perform tasks in each of the following portals. Ensure
you can sign in to each portal, or identify a resource who can make the relevant
configuration changes, before continuing.

| Location                    | Portal URL                                | Task                                                                  |
| --------------------------- | ----------------------------------------- | --------------------------------------------------------------------- |
| D365 Finance & Operations   | _Your F&O environment URL_                | Register MCP clients, verify environment is on v10.0.47               |
| Power Platform admin center | https://admin.powerplatform.microsoft.com | Manage tenant settings, enable Anthropic models, manage environments  |
| Copilot Studio              | https://copilotstudio.microsoft.com       | Create, customize, test, and publish the agent                        |
| Microsoft 365 admin center  | https://admin.microsoft.com               | Manage license assignments and Integrated Apps deployment             |
| Teams admin center          | https://admin.teams.microsoft.com         | Manage Teams app policies and publishing                              |
| Azure Portal                | https://portal.azure.com                  | Optional: Microsoft Entra ID configuration for OAuth app registration |

## Phase 1: Prepare the D365 F&O Environment

The agent depends on the **D365 ERP MCP server (GA)** that ships with F&O
version 10.0.47. You must enable it and register Copilot Studio as an allowed client before
the agent can call any tools.

> **Already enabled?** If the MCP server is already enabled and Copilot Studio is already
> registered as an allowed client, **skip to Phase 2**.

### Step 1-1. Verify Environment Version

1. Sign in to your D365 F&O environment.
2. Go to **Settings (gear icon) → About**.
3. Confirm the platform / application version is **10.0.47** or later. If the environment is
   on an earlier version, update it before continuing. Service updates to the latest version for 10.0.46 or 10.0.45 may need to be applied via **LCS** or the equivalent deployment tooling for 
   Tier-2 / Power Platform  environments.

### Step 1-2. Register Copilot Studio as an Allowed MCP Client

1. In F&O, navigate to **System administration → Setup → Allowed MCP clients**.
2. Click **New**.
3. Register **Copilot Studio** as an app, supplying the application (client) identifier
   exactly as advised in the current product documentation for the preview.
4. Save and close the form.

> ⚠️ Only MCP clients registered in this form can call the D365 ERP MCP server. This is the
> primary access-control surface for the integration.

### Step 1-3. Verify MCP Server Availability (Optional, VS Code)

Delivery engineers who want to validate connectivity before building the agent can connect
from **VS Code** using a local `mcp.json` configuration. This is optional but helpful for
multi-environment testing.

1. In VS Code, create (or update) `.vscode/mcp.json` at the root of your working folder.
2. Add an entry pointing to the D365 ERP MCP server endpoint for your environment, using
   OAuth as the authentication method. (Refer to the current preview documentation for the
   exact endpoint shape.)
3. Reload VS Code and ensure the MCP client surfaces the ERP tools (read / update per
   module).
4. For more information on using MCP server in VS code please refer to 
   this link → https://learn.microsoft.com/en-us/dynamics365/fin-ops-core/dev-itpro/copilot/mcp/mcp-vscode

## Phase 2: Configure Tenant and Environment

### Step 2-1. Enable Anthropic Models

Claude Sonnet 4.6 is the recommended LLM for this agent. You may need to enable it at the
tenant level before it becomes selectable in Copilot Studio.

1. Go to **Power Platform Admin Center** → https://admin.powerplatform.microsoft.com
2. Select **Settings → Tenant settings** (or **Generative AI settings**, depending on the
   current UI).
3. Locate the **Anthropic models** option and enable it for your tenant.
4. Save changes.

### Step 2-2. Select the Agent Environment

1. In **Power Platform Admin Center**, go to **Environments**.
2. Either select an existing environment (same tenant as F&O) or create a new one. A
   dedicated environment for agents is a common pattern.
3. Note the environment name; you will select it in Copilot Studio in Phase 3.

---

## Phase 3: Build the Agent in Copilot Studio

### Step 3-1. Create the Agent

1. Go to **Copilot Studio** → https://copilotstudio.microsoft.com
2. Select the environment created/selected in Phase 2.
3. Click **Create → New agent**.
4. Provide the following:

   | Field       | Value                                                                                                                                                                     |
   | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | Name        | `ERP Configurations Diagnostics Agent`                                                                                                                                    |
   | Description | *Configurations diagnostic Agent for ERP. Analyzes and compares module parameters across D365 F&SCM companies, assigns risk levels, and assists with controlled updates.* |

5. Save the agent.

### Step 3-2. Select the Model

1. Open the agent and navigate to **Settings → Generative AI → Model**.
2. Select **Anthropic Claude Sonnet 4.6**.

> 📌 Claude Sonnet 4.6 is recommended because it performs well on business, finance, and
> supply-chain reasoning tasks relevant to ERP parameter analysis.

### Step 3-3. Set the Agent Instructions

Paste the following into the agent's **Instructions** field (adjust wording to match your
organization's voice if desired):

````markdown
# Purpose
The purpose of this agent is to analyze configuration parameters from Dynamics 365 Finance
and Supply Chain Management (SCM) modules, compare them across multiple companies, and
assess risk levels for each parameter setting.

# General Guidelines
- Maintain a professional and analytical tone.
- Present results in a clear, tabular format.
- Ensure accuracy when retrieving and comparing parameter values.
- Categorize risk levels based on predefined applicability criteria.

# Skills
- Retrieve configuration parameters from D365 Finance and SCM.
- Compare parameter values across multiple companies.
- Perform risk assessment and categorize into low, medium, and high risk.
- Generate tabular reports for easy interpretation.

# Step-by-Step Instructions
## Step 1: Gather Parameters
- Goal: Collect parameter values from each D365 Finance and SCM module.
- Action: Use D365 ERP MCP tools (or Dataverse connectors) to fetch configuration data.
- Transition: Once all parameters are retrieved, proceed to comparison.

## Step 2: Compare Across Companies
- Goal: Identify differences in parameter values between companies.
- Action: Organize data by company and module, then highlight discrepancies.
- Transition: After comparison, move to risk assessment.

## Step 3: Risk Assessment
- Goal: Categorize each parameter setting into low, medium, or high risk.
- Action: Apply predefined rules or thresholds based on parameter applicability and
  business impact.
- Transition: Once risk levels are assigned, prepare the report.

## Step 4: Generate Report
- Goal: Present findings in a tabular format.
- Action: Create a table with columns for Module, Parameter, Company A Value,
  Company B Value, Risk Level, Risk Analysis.
- Transition: Deliver the report to the user.

# Output rules (table format)
- Plain Text
- No HTML (unless the user asks for an HTML report)
- When creating HTML reports on user request add charts and visualization to make it easy 
to analyze the data. Make the html report like a dashboard to provide a overview

# Error Handling and Limitations
- If data retrieval fails, notify the user and suggest checking MCP / API connectivity and
  F&O permissions.
- If risk rules are missing, prompt the user to provide or confirm them.

# Feedback and Iteration
- Ask the user if they need additional filtering or sorting.
- Offer to export the report (Excel, PDF, or HTML) if supported in the current channel.

# Interaction
Example User: "Compare inventory parameters between Company A and Company B."
Agent: "I have retrieved the inventory parameters for both companies. Here is the
comparison table with risk levels assigned."

# Follow-up and Closing
After presenting the report, ask if the user wants to analyze another module, update a
parameter, or export the results.
````

### Step 3-4. Configure Knowledge

1. In the agent, go to **Knowledge → + Add knowledge**.
2. Add the relevant **D365 Learn** site URL(s) so the agent can cite parameter documentation.
   Recommended starting points:
   - Accounts payable parameters overview on learn.microsoft.com
   - Accounts receivable parameters overview on learn.microsoft.com
   - Inventory management parameters overview on learn.microsoft.com
3. **Disable Web search** for the agent. The agent should rely exclusively on the curated
   knowledge source and the MCP server.

### Step 3-5. Connect the D365 ERP MCP Server (Preview)

1. In the agent, go to **Tools → + Add tool**.
2. Select **Model Context Protocol (MCP)** as the tool type.
3. Choose the **Dynamics 365 ERP MCP Server** from the connector gallery (or add
   a custom MCP connection if it is not yet listed).
4. Authenticate using **OAuth**. Sign in with an account that has the required F&O
   permissions for the modules you intend to cover.
5. After connection, verify that the ERP parameter tools are listed under the agent's
   available tools.

> ⚠️ Copilot Studio must be registered in F&O under **Allowed MCP clients** (Phase 1). If
> the connection fails, re-check that registration first.

### Step 3-6. Test the Agent

1. Open the **Test your agent** pane in Copilot Studio.
2. Try the following prompts in order, validating each response:
   - *"Show me all Accounts Receivable parameters and their field values from USMF in a tabular format."*
   - *"Compare the Accounts Payable parameters from USMF, USRT, and USSI. Identify high, medium, and low risk configurations."*
   - *"Update Match Invoice Totals to ON in USRT and confirm the change."* (approve the change when prompted)
3. Verify tabular output, risk ratings, and that updates are only performed after your
   explicit confirmation.

## Phase 4: Publish the Agent

### Step 4-1. Configure Channels

The agent can be published to any Microsoft channel. Teams and SharePoint are the most
common.

1. In Copilot Studio, open the agent and go to **Channels**.
2. Enable **Microsoft Teams**.
   - Provide an icon and short description.
   - Save and publish.
3. (Optional) Enable **SharePoint** if your organization wants the agent available on a
   SharePoint site.

### Step 4-2. Make the Agent Available to Users

1. In the **Microsoft 365 admin center**, go to **Settings → Integrated apps**.
2. Locate the agent, click **Deploy**, and target the appropriate users or groups.
3. (Optional) In the **Teams admin center**, verify the relevant Teams app permission and
   setup policies allow the agent.

### Step 4-3. Connect Multiple Environments (Optional)

For delivery engineers supporting several customer environments, you can connect additional
D365 F&O environments:

- **In Copilot Studio**: add additional MCP tool connections, each pointing at a different
  F&O environment, and name them clearly (Dev / Test / Prod or customer-specific).
- **In VS Code**: maintain a `mcp.json` file listing each environment's MCP endpoint for
  local testing and validation.

## Phase 5: Operate and Iterate

### Step 5-1. Run Your First Diagnostic Review

Recommended first-run flow for a new implementation:

1. Ask the agent: *"Show me all [Module] parameters from [Primary Company] in a tabular
   format. Include risk analysis and an implementation plan."*
2. Ask: *"Compare [Module] parameters across [Company A], [Company B], [Company C] and
   highlight high/medium/low risks."*
3. Review the output with the functional team.
4. Use the prioritized recommendations to drive parameter updates (either manually or via
   the agent's update flow).

### Step 5-2. Apply Updates via the Agent

1. Phrase the request explicitly: *"Update `<parameter name>` from `<current value>` to
   `<new value>` in `<company>` and confirm."*
2. When the agent asks for confirmation, verify the parameter name, old value, new value,
   and target company.
3. Approve. The agent calls the MCP update tool and returns a status.
4. Follow up with: *"What are the remaining recommendations for `<company>`?"*

### Step 5-3. Keep the Knowledge Base Current

- When new modules are exposed by the D365 ERP MCP server, re-test the agent's ability to
  discover and use them without code changes.
- Update the knowledge source URLs when D365 Learn pages are reorganized.
- Revisit the risk rubric in the agent instructions each time controls guidance (e.g., SOX
  templates, internal policy) is updated.

## Troubleshooting

| Symptom                                | Likely Cause                                                             | Resolution                                                                                       |
| -------------------------------------- | ------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------ |
| Agent cannot reach the MCP server      | Copilot Studio not registered in F&O's **Allowed MCP clients**           | Register it in System Administration → Setup → Allowed MCP clients                               |
| Preview tools not visible in the agent | Preview feature not enabled in F&O                                       | Enable *(Preview) Dynamics 365 ERP Model Context Protocol server* in Feature management          |
| Claude Sonnet 4.6 not selectable       | Anthropic models not enabled in tenant                                   | Enable in Power Platform Admin Center tenant settings                                            |
| Parameter read fails                   | Signed-in user lacks F&O permission for the module or legal entity       | Adjust the user's security roles in F&O                                                          |
| Update fails silently                  | F&O record is locked by another session or parameter requires a workflow | Retry; if persistent, apply the change manually and confirm with the agent                       |
| Agent returns stale values             | F&O cached the parameter                                                 | Re-run the diagnostic query; ensure the environment has been refreshed after any platform update |
