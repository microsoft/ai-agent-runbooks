# Sample Prompts and Scenario Catalog

> The Dynamics 365 ERP Configurations Diagnostics Agent ships with **no hard-coded query
> library**. Instead, it discovers capabilities dynamically from the **D365 ERP MCP server
> (Preview)**, which exposes module parameters and update actions as tools the LLM can call.
>
> The prompts below are grouped by the typical diagnostic, comparison, and update flows that
> an implementation team uses during a D365 F&O rollout. Any module exposed by the MCP server
> can be targeted with the same pattern, swap the module name and the company.
>
> As the MCP server's coverage expands, the prompt catalog grows automatically. No code
> changes are required in the agent.

---

> **⚙️ First-Time Setup**
>
> Before running any prompt, confirm the following are in place (see [3.Runbook.md](3.Runbook.md)):
>
> - D365 F&O environment is on **version 10.0.47** (Tier-2 or Power Platform environment).
> - The *(Preview) Dynamics 365 ERP Model Context Protocol server* feature is **enabled** in
>   Feature management.
> - **Copilot Studio** is registered under **System Administration → Setup → Allowed MCP clients**.
> - The agent's MCP tool connection is authenticated via **OAuth** with an account that has
>   read (and, when applicable, update) access to the relevant modules.

## How to Use These Prompts

- Copy any prompt below and use it directly in a chat with the agent.
- You can interact with the agent through:
  - **Microsoft Teams** (via *Built for your org* after org-wide deployment)
  - **SharePoint** (if the agent is published to a SharePoint site)
  - **Copilot Studio Test pane** (for authors and delivery engineers)
- Prompts are flexible. Claude Sonnet 4.6 matches intent, so you do not need to use the
  exact wording.
- Refine results by specifying: **module**, **company / legal entity**, **parameter**,
  **new value**, or **output format** (table, HTML, Excel-ready).

# D365 Finance & Operations Prompts

The categories below reflect the typical implementation lifecycle: diagnose each module,
compare across companies, assess risk, plan remediation, and apply controlled updates.

## Category 1: Single-Company Module Diagnosis

Retrieve every parameter for a module in a specific legal entity, and get an
immediate risk read on each setting.

| #   | Prompt                                                                                                                                                                              | What It Returns                                                                     |
| --- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------- |
| 1   | Show me all Accounts Receivable parameters and their field values from USMF in a tabular format. Include risk analysis and an implementation plan for any missing parameter values. | Table of all AR parameters with values, Risk Level (🔴/🟡/✅), and recommended actions |
| 2   | Show me all Accounts Payable parameters in USMF. Group the results by section (General, Vendor Management, Invoice, Settlement, etc.).                                              | Sectioned table of AP parameters with values                                        |
| 3   | Show me all Inventory Management parameters from USMF along with the field values. Identify high, medium, and low risk configurations with their impact.                            | Inventory parameter table with risk rating and impact description                   |
| 4   | Show me all General Ledger parameters in USMF with risk analysis.                                                                                                                   | GL parameter table with risk analysis                                               |
| 5   | Show me all Procurement & Sourcing parameters in USMF and flag any missing controls.                                                                                                | Procurement parameter table with control-gap flags                                  |
| 6   | Show me all Sales & Marketing parameters in USMF.                                                                                                                                   | Sales parameter table                                                               |

### Example Interaction

> **User**: Show me all Accounts Receivable parameters and their field values from USMF in a tabular format. Show me the risk analysis and implementation plan for addressing any missing parameter values.
>
> **Agent**: Here are the Accounts Receivable parameters for **USMF**:
>
> | Parameter | Value | Risk Level | Risk Analysis |
> |---|---|---|---|
> | Credit limit type | None | 🔴 HIGH | No credit limit checking. Risk of extending credit beyond customer capacity. |
> | Post packing slip in ledger | Yes | ✅ LOW | Standard posting for inventory valuation accuracy. |
> | Check credit limit on sales order | Balance only | 🟡 MEDIUM | Partial check. Consider **Balance + all** for tighter control. |
>
> **Implementation Plan**
>
> - **Immediate (1 week)**: Enable credit limit type (Balance + All).
> - **Short-term (1 month)**: Review packing slip posting alignment across all legal entities.
>
> Would you like me to compare these against USRT and USSI, or apply an update?

## Category 2: Cross-Company Comparison

Compare the same module across two or more legal entities to surface configuration drift,
missing controls, and inconsistencies.

| #   | Prompt                                                                                                                                          | What It Returns                                      |
| --- | ----------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------- |
| 7   | Compare the Accounts Payable parameters from USMF, USRT, and USSI. Identify high, medium, and low risk configurations as part of this analysis. | Side-by-side AP comparison with per-row Risk Level   |
| 8   | Compare inventory management parameters between USMF and USRT. Highlight discrepancies only.                                                    | Table of differing parameters with risk rating       |
| 9   | Compare General Ledger parameters across USMF, USRT, and USSI. Call out any settings that violate three-way matching or SOX expectations.       | Comparison with SOX / control callouts               |
| 10  | Compare Procurement & Sourcing parameters across USMF, USRT, and USSI. Focus on approval workflows and matching policies.                       | Comparison focused on approval / matching parameters |
| 11  | Compare Accounts Receivable parameters between USMF and USRT. Show only parameters that differ.                                                 | Differences-only AR comparison                       |
| 12  | Compare vendor approval workflow settings across USMF, USRT, and USSI.                                                                          | Workflow-focused comparison with risk rating         |

### Example Interaction

> **User**: Compare the Accounts Payable parameters from USMF, USRT, and USSI. Identify high, medium, and low risk configurations as part of this analysis.
>
> **Agent**: Here is the comparison of **Accounts Payable** parameters across **USMF**, **USRT**, and **USSI**.
>
> **Vendor Approval & Workflow**
>
> | Parameter | USMF | USRT | USSI | Risk Level | Risk Analysis |
> |---|---|---|---|---|---|
> | Enable Vendor Approvals | Yes | No | No | 🔴 HIGH | Critical difference. USRT and USSI have no vendor approval workflow, creating control gaps for vendor master data changes. |
> | Data Entity Behavior | Create change proposals | Allow without approval | Allow without approval | 🔴 HIGH | USRT and USSI allow vendor changes via data entities without approval, major fraud risk. |
>
> **Invoice Validation & Matching**
>
> | Parameter | USMF | USRT | USSI | Risk Level | Risk Analysis |
> |---|---|---|---|---|---|
> | Enable Invoice Matching Validation | Yes | Not checked | Not checked | 🔴 HIGH | Invoice matching is a key control to prevent overpayments and fraud. |
> | Line Matching Policy | Three-way matching | Not checked | Not checked | 🔴 HIGH | Three-way matching (PO, receipt, invoice) is best practice. |
>
> **Top Findings**
>
> 1. **USRT & USSI lack vendor approval workflows**, enable to match USMF.
> 2. **Invoice matching verification pending for USRT & USSI**, confirm and enable three-way matching.
> 3. **Bank account change controls missing across all companies**, enable approval for create and update.
>
> Would you like the full **Immediate / Short-term / Medium-term** action plan, or should I apply one of the fixes now?

## Tips for Getting the Best Results

- **Be specific about the module and company.** *"Accounts Payable in USMF"* works better than *"AP parameters"*.
- **Ask for risk analysis explicitly** when you want the 🔴 / 🟡 / ✅ rating and justification.
- **Chain prompts**: diagnose → compare → plan → update. The agent preserves context in a conversation.
- **For updates, include the target value and company in the same prompt** to minimize back-and-forth.
- **Request HTML output** when you want a shareable artifact for audit or steering committee reviews.
