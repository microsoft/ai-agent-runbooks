💡 Specific Recommendations

🔴 [HIGH] Upgrade Line Matching Policy to Three-Way Match USRT

Issue: USRT uses TwoWayMatch, only comparing PO quantity vs. invoice quantity. USMF correctly uses ThreeWayMatch (PO + Product Receipt + Invoice).

Impact: Without three-way matching, USRT can pay invoices for goods not yet received — a critical fraud and financial loss risk.

  • Navigate to: AP → Setup → AP Parameters → Invoice matching → Line matching policy
  • Change value from TwoWayMatch → ThreeWayMatch
  • Review existing vendor-specific matching overrides
  • Validate that product receipts are consistently posted before invoicing

🔴 [HIGH] Require Approval for Invoice Match Discrepancies USRT

Issue: USRT is set to AllowWithWarning, which permits posting invoices with matching discrepancies with just a warning. USMF correctly uses RequireApproval.

Impact: Invoices with price or quantity discrepancies can be posted without oversight, risking unauthorized payments.

  • Navigate to: AP → Setup → AP Parameters → Invoice matching → Post invoice match discrepancies
  • Change value from AllowWithWarning → RequireApproval
  • Set up an approval workflow for discrepancy review
  • Train AP staff on the discrepancy resolution process

🔴 [HIGH] Enable Charge Matching USRT

Issue: USRT has Match Charges = No, meaning freight, handling, and other charges on invoices are never validated against POs.

Impact: Vendors can inflate charges without detection. USMF correctly matches charges.

  • Navigate to: AP → Setup → AP Parameters → Invoice matching → Match charges
  • Set Match charges to Yes
  • Define charge tolerance thresholds (currently 0% — consider 2–5% tolerance)
  • Review and update vendor charge agreements

🔴 [HIGH] Configure Prepayment Invoice Posting Profile USMF

Issue: USMF has no Prepayment Invoice Posting Profile set. USRT correctly uses "PRE".

Impact: Without this profile, prepayment invoices may post to incorrect ledger accounts, causing financial misstatement.

  • Navigate to: AP → Setup → AP Parameters → Ledger and sales tax → Prepayment invoice posting profile
  • Assign the PRE posting profile (verify it exists in Vendor posting profiles)
  • Run a historical prepayment report to identify any mismapped entries
  • Coordinate with GL team to confirm account assignments

🟠 [MEDIUM] Enable Product Receipt Matching Automation USRT

Issue: USRT has Automation – Product Receipt Matching = No. USMF benefits from automated receipt-to-invoice matching.

Impact: Manual matching increases processing time, human error, and delays invoice approval.

  • Enable: AP → Setup → AP Parameters → Invoice automation → Automate product receipt matching
  • Set max attempts to 24 (already aligned with USMF)
  • Also enable Automation – Matching Prior to Workflow for end-to-end automation

🟠 [MEDIUM] Enable Vendor Change Proposal Workflow USRT

Issue: USRT has Change Proposal = No. USMF enforces a change proposal workflow for vendor master modifications.

Impact: Without change control, unauthorized updates to vendor bank accounts or contact details could enable fraud (e.g., BEC — Business Email Compromise attacks).

  • Enable: AP → Setup → AP Parameters → Vendor → Enable change proposals
  • Configure a workflow for change approval (minimum 2 approvers)
  • Segregate duties: requester ≠ approver

🟠 [MEDIUM] Keep Sales Tax Adjustments for PO Invoices USRT

Issue: USRT has KeepSalesTaxAdjustmentsForPOInvoices = No, meaning manual tax adjustments on PO-based invoices are discarded.

Impact: Tax compliance risk — manual corrections made by AP staff are lost on posting, potentially leading to incorrect tax reporting.

  • Navigate to: AP → Setup → AP Parameters → Invoice → Keep sales tax adjustments
  • Set to Yes to preserve tax adjustments
  • Review open PO invoices for any tax discrepancies

🟠 [MEDIUM] Enable Invoice Validation in Journal & W-9 Validation BOTH

Issue: Both companies have ValidateInvoicesInTheJournal = No and W-9 Validation = No.

Impact: Invoices can be entered in journals without validation. W-9 non-compliance creates IRS 1099 reporting risks for US entities.

  • Enable journal validation: AP Parameters → Invoice → Validate invoices in the journal
  • Enable W-9 validation for US vendors: AP Parameters → Vendors → W-9 validation
  • Review existing vendor W-9 records for completeness
