# Dynamics 365 Autonomous PO Generation Agent - Overview

| Property | Value |
|----------|-------|

| **Agent Type** | Autonomous agent |
| **Primary Tools** | Copilot Studio, Dynamics 365 F&O, Power Automate, Outlook |

| **Complexity** | Intermediate |
| **Status** | ✅ Available |

This scenario outlines how to deploy and configure the PO Agent, an AI agent that captures purchase request details via email or Teams from an attached PO request. The agent takes information from the request and creates a drafted purchase order that gets sent back to the requester, via email or teams along with a summary of any information that may have been missing. The goal is to reduce manual effort, improve data consistency, and accelerate the purchasing process for frontline workers who need to quickly formulate a PO to complete their work.

The agent is also able to answer questions and make reports around the Dynamics 365 F&O platform for Procurement team members who access it directly. This designed to standardize intake, validate required information, and prepare purchase orders in a repeatable way across request channels. This will allow the Procurment team to have a faster way to learn about vendors and get more information on PO request trends and previously submitted PO requests to expedite their approval process.

## Problem Statement

Frontline workers often need to submit purchase orders to get the supplies they need for their work and procurement teams (PO approvers) often need information on vendors and previous POs to ensure that all the information on the order is correct and that they are able to make the purchase. Because the required details are frequently incomplete, inconsistent, or buried in unstructured communications, staff must spend significant time collecting information, validating vendor and line-item details, and manually entering requests into business systems. This process is repetitive, slow, and prone to human error, which can delay approvals, create rework, and reduce visibility into purchasing activity. Organizations need a more reliable way to capture purchase request information, standardize it, and generate draft purchase orders efficiently while still preserving human review and control before submission.

## Solution Summary

The Dynamics PO Generator is an autonomous agent built in Microsoft Copilot Studio to automate intake and draft creation of purchase orders in Dynamics 365. It monitors a designated Outlook mailbox, reads incoming purchase request emails, extracts data from attached Word or PDF documents, and submits the request to Dynamics 365 through the MCP server for draft PO creation.

The agent operates with partial autonomy: it can independently trigger on qualifying incoming emails, retrieve attachments, extract structured purchase request data, validate key fields such as company, vendor, requestor, buyer, facility, amounts, and dates, and create the purchase order in draft status. Human review remains in the process before approval, helping the organization reduce manual effort and improve consistency without removing human oversight.

The solution combines Copilot Studio, Power Automate, Outlook, document extraction logic, and Dynamics 365 integration to orchestrate end-to-end processing. It also sends a detailed reply email summarizing the created PO, validation issues, and assumptions made, and can be extended with Dataverse audit logging to improve traceability, monitoring, and operational support as the proof of concept matures. Though the fully autonomous flow fits most use cases, this runbook also covers a manual flow via Teams that can be used to accomplish the PO generation as well and the scenario where the procurment team just wants to get reports done or questions answered via the agent directly.

## Key Capabilities

| Capability | Description |
|-----------|-------------|
| **Copilot Agent** | The agent will facilitate the intake and processing of purchase requests. Triggered by incoming emails to a common Outlook mailbox tracked by the agent, it will read incoming PO request email, extract data from attachment and send to D365 using MCP Server. |
| **Dynamics 365 PO Creation** | Using D365 MCP Server, the agent will create and update purchase order entries in the F&O system using data from email attachment in Word or PDF format. |
| **Validation** | Agent performs validation of the key fields like Company, Vendor, Requestor, Buyer, Facility, Amounts and Dates before creating the PO in D365. The PO is generated in draft status for human review before approving. |
| **Logging** | Agent sends a detailed log of the created PO, with information about any validation check errors, assumptions made, by a reply email to the requester. |

## How It Works

The agent is triggered through Power Automate when a new email arrives trigger with configured filtering conditions such as subject-based rules. Any changes to trigger behavior must be made in the flow.

The solution includes several tools and integrations, such as:

- Dynamics 365 ERP MCP for PO creation
- Attachment retrieval for incoming documents
- Prompt logic to extract structured data from Word/PDF files
- Additional orchestration for email handling, user resolution, and document processing

The agent also uses knowledge sources in Copilot Studio to improve understanding of D365 concepts and response accuracy.

### Extensibility & Maintenance

As currently designed, the Autonomous PO Agent provides operational visibility primarily through Dynamics 365 draft purchase orders and requester reply emails. If teams need deeper insight into agent execution itself—such as trigger history, extraction outcomes, validation failures, duplicate handling, or downstream flow errors—they should extend the solution with dedicated logging and monitoring, for example by writing run details and exception states to Dataverse, SharePoint, or another operational store and surfacing alerts through Power Automate.

## In Scope / Out of Scope

### ✅ In Scope
- Monitoring a designated Outlook mailbox for qualifying purchase request emails based on configured trigger conditions 

- Retrieving and processing supported request attachments, such as Word documents and PDFs, to extract purchase order details 

- Validating required fields including company, vendor, requestor, buyer, facility, amounts, and dates before draft creation 

- Creating purchase orders in Dynamics 365 through the MCP integration in draft status for human review 

- Sending requester-facing response emails that summarize PO creation results, validation issues, and assumptions made during processing 

- Proper handling of multiple purchase order requests submitted in one email (each separate request will lead to a separate purchase order)

### ❌ Out of Scope

- Having agent validate pre-conditions for PO creation

- production deployment or production-grade monitoring

- SLA-backed uptime guarantees

- UI/UX enhancements

- Automatic approval or final submission of purchase orders without human review 

- Direct processing of request channels outside the configured workflow, unless additional integrations are explicitly implemented 

- Creation or maintenance of vendor master data, buyer records, or other upstream ERP reference data 

- Resolution of incomplete, contradictory, or policy-exception requests beyond basic validation and requester notification 

- Custom model training or fine-tuning for document extraction beyond the configured solution design 

- Advanced exception-management patterns such as full retry orchestration, dead-letter queues, or duplicate-resolution workflows unless separately designed 

- Broader procurement transformation activities, including downstream invoicing, goods receipt, or sourcing process redesign
