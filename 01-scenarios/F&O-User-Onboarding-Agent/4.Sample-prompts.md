# Sample Prompts — F&O User Onboarding Agent

> The **FOUserOnboardingAgent** supports three direct conversational prompts.
> All other interactions (user details, role selection, approvals, confirmations)
> are driven through **Adaptive Cards** returned by the agent's flows.

---

## Prompt 1: Onboard User

**Trigger phrase:**
```
Onboard user
```

**What happens:** The agent launches the onboarding workflow and presents an Adaptive Card
to collect the new hire's details (name, email, company, security roles). The rest of the
flow — including approval (Mode 1) and D365 F&O provisioning — proceeds via Adaptive Cards.

**Example:**

> **Admin**: Onboard user
>
> **Agent**: *(returns Adaptive Card in Teams/M365 Copilot)*
>
> **New D365 F&O User Onboarding Request**
>
> | Field | Input |
> |---|---|
> | Full Name | *(text field)* |
> | Email Address (UPN) | *(text field)* |
> | D365 F&O Company | *(dropdown — e.g., USMF, GBSI)* |
> | Security Roles | *(multi-select or text field)* |
>
> **[Submit Request]**

---

## Prompt 2: Security Posture of the User

**Trigger phrase:**
```
Security posture of the user
```

**What happens:** The agent presents an Adaptive Card asking for the user's email address,
then queries D365 F&O and returns the user's current security role assignments.

**Example:**

> **Admin**: Security posture of the user
>
> **Agent**: *(returns Adaptive Card)*
>
> **Check Security Posture**
>
> | Field | Input |
> |---|---|
> | User Email (UPN) | *(text field — e.g., john.smith@contoso.com)* |
>
> **[Check]**
>
> *(After submission)*
>
> **Security Posture for john.smith@contoso.com**
>
> | Company | Assigned Security Roles |
> |---|---|
> | USMF | Accounts Payable Manager, Expense Report Reviewer |

---

## Prompt 3: Offload User

**Trigger phrase:**
```
Offload user
```

**What happens:** The agent presents an Adaptive Card to collect the departing user's
details, then triggers the offboarding/deprovisioning workflow in D365 F&O.

**Example:**

> **Admin**: Offload user
>
> **Agent**: *(returns Adaptive Card)*
>
> **D365 F&O User Offboarding Request**
>
> | Field | Input |
> |---|---|
> | User Email (UPN) | *(text field — e.g., jane.doe@contoso.com)* |
> | D365 F&O Company | *(dropdown — company to remove access from)* |
> | Reason | *(text field — e.g., Resignation, Contract end)* |
>
> **[Submit Offboarding Request]**

---

## Notes

- All three prompts are the **only supported entry points** for the agent
- Phrases other than the exact triggers above may not be recognized — instruct users to use the exact wording
- After submitting any Adaptive Card, the agent handles the rest of the workflow automatically
- For approval flows (Mode 1), approvers respond directly via the Adaptive Card posted in the **FNOUserApprove** Teams channel — no additional prompts are needed
