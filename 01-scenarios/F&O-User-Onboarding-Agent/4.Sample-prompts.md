# Sample Prompts — F&O User Onboarding Agent

> These sample prompts are designed to help HR admins, IT admins, and delivery engineers
> get the most out of the **FOUserOnboardingAgent**.
> Prompts are organized by category based on common onboarding scenarios.

---

## How to Use These Prompts

- Start a conversation with the **FOUserOnboardingAgent** in **Microsoft Teams** or **Microsoft 365 Copilot**
- Use the prompts below to initiate, guide, or test the onboarding workflow
- The agent will guide you through a structured form to collect all required information
- Feel free to modify the prompts to match your organization's naming conventions

---

## Category 1: Initiating an Onboarding Request

These prompts start the onboarding conversation and launch the guided form.

| # | Prompt | What the Agent Does |
|---|---|---|
| 1 | `I need to onboard a new F&O user` | Starts the guided onboarding form and collects new hire details |
| 2 | `Set up a new Dynamics 365 Finance user for my team` | Initiates onboarding workflow and asks for user details |
| 3 | `Create a new user in F&O for Jane Doe` | Begins the form, pre-fills name context, collects remaining fields |
| 4 | `Provision access to Finance and Operations for a new employee` | Starts the full provisioning workflow |
| 5 | `I have a new hire starting Monday and need their D365 access set up` | Triggers onboarding flow with urgency context |

### Example Interaction

> **Admin**: I need to onboard a new F&O user
>
> **Agent**: I can help with that. Let me collect the details for the new user.
> Please provide the following information:
>
> 1. **Full name** of the new hire
> 2. **Email address** (their Microsoft 365 UPN)
> 3. **D365 F&O Company** (legal entity code, e.g., USMF, GBSI)
> 4. **Security roles** to assign (e.g., Accounts Payable Manager, Inventory Manager)
>
> You can start by providing the full name.

## Category 3: Approval Workflow Interactions (Mode 1 Only)

These prompts apply when the agent is configured in **Mode 1** (CreateADUser=1, Approval=Yes).
The approver receives a Teams Adaptive Card and responds directly in the Teams channel.

| # | Scenario | Action |
|---|---|---|
| 11 | Approver receives Adaptive Card in Teams | Click **Approve** to proceed with user creation and D365 provisioning |
| 12 | Approver wants to reject the request | Click **Reject** — agent returns rejection notice to the requestor |
| 13 | Approver has questions before approving | Reply in the Teams channel; contact requestor outside the agent flow |
| 14 | Requestor wants to check approval status | Type `What is the status of my onboarding request for John Smith?` |
| 15 | Requestor wants to cancel before approval | Type `Cancel the onboarding request for John Smith` |

### Example Adaptive Card Content (Teams Channel)

> **FOUserOnboardingAgent** posted in **#FNOUserApprove**:
>
> **New D365 F&O User Provisioning Request**
>
> | Field | Value |
> |---|---|
> | New User | John Smith |
> | Email | john.smith@contoso.com |
> | Company | USMF |
> | Roles | Accounts Payable Manager, Expense Report Reviewer |
> | Requested By | HR Admin |
> | Request Time | 2026-04-13 09:32 UTC |
>
> **[Approve]** &nbsp;&nbsp; **[Reject]**

## Category 5: Error and Exception Handling

These prompts help admins understand and respond to error scenarios.

| # | Scenario | What to Do |
|---|---|---|
| 20 | Agent returns: "AD User not found. Cannot proceed." | This is Mode 2 behavior — the Entra ID user does not exist. Work with IT to create the account first, then retry. |
| 21 | Agent returns: "Provisioning failed — company code not recognized" | Verify the legal entity code is correct in D365 F&O (e.g., check that `USMF` exists in your environment) |
| 22 | Agent returns: "Security role not found" | Confirm the role name matches exactly as it appears in D365 F&O System Administration → Security → Roles |
| 23 | Teams Adaptive Card did not appear | Verify `TeamName` and `ChannelName` in `OnboardAgentConfig` are correct |
| 24 | Agent confirms creation but user not in D365 | Check the D365 FO Provision Flow run history in Power Automate for errors |

### Example Error Interaction (Mode 2 — User Not Found)

> **Admin**: Onboard user alex.johnson@contoso.com in USMF with Inventory Manager role
>
> **Agent**: I checked Microsoft Entra ID for alex.johnson@contoso.com.
>
> ⚠️ **AD User not found. Cannot proceed.**
>
> The agent is configured in validation-only mode (CreateADUser = 0). The user must
> exist in Entra ID before provisioning can proceed in D365 F&O.
>
> **Next steps**:
> 1. Create the Entra ID user account manually (Microsoft 365 Admin Center or Azure Portal)
> 2. Assign the appropriate Microsoft 365 license
> 3. Re-run this onboarding request after the account is created

## Tips for Getting Better Results

| Tip | Example |
|---|---|
| **Be specific with company codes** | Instead of *"the US company"*, use the legal entity code: *"USMF"* |
| **List roles explicitly** | *"Assign Accounts Payable Manager and Expense Report Reviewer"* is better than *"standard AP roles"* |
| **Use exact email UPN** | Ensure the email matches the expected Microsoft 365 UPN format: `firstname.lastname@yourdomain.com` |
| **Confirm before submitting** | When the agent shows the confirmation summary, review all details before saying *"Yes, proceed"* |
| **Use the cancel prompt if needed** | If you need to stop mid-flow: *"Cancel this request"* |

## Out-of-Scope Topics

The **FOUserOnboardingAgent** is designed for user provisioning workflows only.
If a request falls outside this scope, the agent will respond with a fallback message:

> *"I can only assist with D365 F&O user onboarding. For other requests, please contact your IT helpdesk."*

Topics that are out of scope include:

- Deprovisioning or removing user access
- Password resets or account unlocks
- D365 F&O data queries or business process support
- License management beyond initial user creation
- Onboarding to platforms other than Dynamics 365 Finance & Operations
