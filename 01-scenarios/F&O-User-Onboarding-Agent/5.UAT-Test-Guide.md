# UAT Test Guide ŌĆö F&O User Onboarding Agent

> **Document Version**: 1.0
> **Last Updated**: April 2026
> **Classification**: Confidential ŌĆö Internal Use Only
> **Scope**: Functional User Acceptance Testing (UAT) for all three operational modes

---

## Overview

This guide provides the complete set of User Acceptance Test (UAT) scripts for the
**F&O User Onboarding Agent**. The tests are organized by the three operational modes
supported by the agent, each controlled by the `OnboardAgentConfig` settings in Dataverse.

All three test guides must be executed before go-live to validate that the agent behaves
correctly under each mode, handles both success and error paths, and produces the expected
outcome in Dynamics 365 Finance & Operations.

---

## Test Guide 01 ŌĆö Mode 1: CreateADUser=1 | Approval Required=Yes

### Purpose

Validates the full end-to-end onboarding flow with **automatic Entra ID user creation**
and a **Teams Adaptive Card approval step** before D365 F&O provisioning occurs.

### Configuration Required

Before running this test, set the following values in `OnboardAgentConfig`:

| Column | Value |
|---|---|
| `CreateADUser` | `1` |
| `ApprovalRequired` | `Yes` |
| `TeamName` | Your test Teams team name (e.g., `IT Operations`) |
| `ChannelName` | `FNOUserApprove` |
| `GraphClientID` | Application (client) ID from App Registration |
| `GraphClientSecret` | Client secret value |
| `GraphTenantID` | Directory (tenant) ID |

### Required Graph API Permissions

| Permission | Type | Purpose |
|---|---|---|
| `User.ReadWrite.All` | Application | Create Entra ID user accounts |
| `Directory.ReadWrite.All` | Application | Assign directory roles |

> Ō£ģ Admin consent must be granted for both permissions before testing.

---

### Test Scenario 01-A ŌĆö Happy Path (New User, Approved)

**Objective**: Validate that a new user is created in Entra ID, approval is routed to
Teams, and user is provisioned in D365 F&O after approval.

#### Test Data

| Field | Value |
|---|---|
| New Hire Name | `CAF User` (or your test user name) |
| Email | `CAF.user@[yourtenant].onmicrosoft.com` |
| Company | `USMF` (or your test company code) |
| Security Role | `Accounts Payable Manager` (or your test role) |
| Approval Action | **Approve** |

#### Test Steps

| # | Step | Action | Expected Result |
|---|---|---|---|
| 1 | Open the agent | Go to Teams or M365 Copilot ŌåÆ open FOUserOnboardingAgent | Agent loads and greets with welcome message |
| 2 | Initiate onboarding | Type: `I need to onboard a new F&O user` | Agent starts the guided form |
| 3 | Provide new hire name | Enter: `CAF User` | Agent acknowledges and asks for email |
| 4 | Provide email | Enter: `CAF.user@[yourtenant].onmicrosoft.com` | Agent acknowledges and asks for company |
| 5 | Provide company | Enter: `USMF` | Agent acknowledges and asks for security roles |
| 6 | Provide role | Enter: `Accounts Payable Manager` | Agent shows confirmation summary |
| 7 | Confirm submission | Type: `Yes, proceed` | Agent submits request and triggers flow |
| 8 | Entra ID check | (Automatic ŌĆö observe flow run) | Flow confirms user does not exist in Entra ID |
| 9 | Entra ID creation | (Automatic ŌĆö observe flow run) | Flow creates user: `statusCode: 200`, `isusercreated: 1` |
| 10 | Teams Adaptive Card | Check Teams `FNOUserApprove` channel | Adaptive Card appears within 60 seconds with correct user details |
| 11 | Approve request | Approver clicks **Approve** on the card | Approval response sent to flow |
| 12 | D365 F&O provisioning | (Automatic ŌĆö observe flow run) | User provisioned in USMF with Accounts Payable Manager role |
| 13 | Confirmation | Check agent chat | Success message returned with provisioned user details |
| 14 | Verify in D365 F&O | Log in to D365 F&O ŌåÆ System Administration ŌåÆ Users | User `CAF User` appears with correct company and role |

#### Expected Outcomes

| Outcome | Expected Value | Actual Value | Pass/Fail |
|---|---|---|---|
| Graph API response | `statusCode: 200`, `isusercreated: 1` | | ŌśÉ |
| Teams Adaptive Card displayed | Card visible in `FNOUserApprove` channel | | ŌśÉ |
| Approval processed | Flow resumes after Approve click | | ŌśÉ |
| User in D365 F&O | User appears in target company with correct roles | | ŌśÉ |
| Confirmation message | Success message in agent chat | | ŌśÉ |

## Test Guide 02 ŌĆö Mode 2: CreateADUser=0 | Validation Only

| `GraphClientID` | Application (client) ID from App Registration |

### Required Graph API Permissions

| Permission | Type | Purpose |
|---|---|---|
| `User.Read.All` | Application | Read-only validation ŌĆö check user existence |

> ŌÜĀ’ĖÅ Do NOT grant `User.ReadWrite.All` for Mode 2 testing ŌĆö validate that read-only permission is sufficient.
