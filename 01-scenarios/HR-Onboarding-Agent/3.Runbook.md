# Step-by-Step Guide — HR Onboarding Autonomous Agent

> **Scenario**: HR Self-Help Autonomous Assistant  
> **Platform**: Microsoft Copilot Studio + Power Platform + ServiceNow  
> **Target Readers**: CSA / Vendor / PM  
> **Last Updated**: March 19, 2026

---

## Overview

New hires often face challenges understanding company policies, accessing
necessary resources, and integrating into the team. This agent streamlines
the onboarding process by:

- Answering HR-related questions using a ServiceNow knowledge base
- Automatically responding to email inquiries sent to an HR mailbox
- Being accessible via Microsoft Teams and M365 Copilot

---

## Prerequisites

Before starting, confirm the following are in place:

| Requirement | Details | Where to Configure |
|---|---|---|
| Microsoft 365 Copilot License | Required per user for M365 Copilot channel access | M365 Admin Center |
| Power Platform Developer Environment | Free; must be non-default with Dataverse enabled | Power Platform Admin Center |
| ServiceNow Developer Tenant | Free developer instance for knowledge base setup | developer.servicenow.com |
| Copilot Extensibility Enabled | Allow users access to Copilot agents | M365 Admin → Integrated Apps |
| Teams Custom App Side-loading | Required for Teams channel deployment | Teams Admin Center |

### Step 2-5. Update Agent Instructions

1. On the Overview page, click **"Edit"** next to **Instructions**
> ![Edit instructions](https://github.com/microsoft/cloud-accelerate-factory/blob/bb5b1767b3056ea87bbf412ee5b7e66297dbe5b1/01-scenarios/HR-Onboarding-Agent/0.Resources/Images/074.png)

   > ⚠️ Replace `[ServiceNowKB]` with your actual knowledge name (e.g., `ServiceNowKB5`)  
   > ⚠️ Replace `[SendEmail]` with your tool name. Use `/` to select a Tool inline.

   ```
   # Capabilities:
   This is an HR onboarding agent which can provide answers for new hires onboarding questions

   ## HR related questions:
   * use "[ServiceNowKB]" knowledge to answer the questions.
   * If no result is found from "[ServiceNowKB]", just reply "I don't know,
     please reach out to HRInfo@[your-tenant].onmicrosoft.com".
   * Avoid providing content which is not from provided knowledge "[ServiceNowKB]".

   ## Email trigger
   the agent can be triggered by an email event, use the following logics to process:
   * The question is stored inside the email "Body" property
   * Use "[ServiceNowKB]" to find out the answer
   * When information is found, use email defined in "From" property from email
     trigger to send email via "[SendEmail]" tool. the email includes "Subject"
     which includes the overview of question and "Body" includes the information
     found from knowledge. the "Body" format is HTML format.
   ```
