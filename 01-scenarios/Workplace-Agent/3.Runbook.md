## Runbook Overview

This runbook provides step-by-step deployment instructions for the **Workplace Agent** — a Microsoft Copilot Studio-based agent that handles internal FAQ inquiries and application request workflows using SharePoint, Dataverse, and Power Automate.

| Attribute | Details |
|-----------|---------|
| **Audience** | CSA, Vendor Engineer, IT Administrator, AI Promotion Team |
| **Difficulty** | Intermediate |
| **Est. Duration** | 3–5 hours (Basic); +2–4 hours for Advanced Azure Extension |
| **Agent Type** | Declarative Agent (Microsoft Copilot Studio, Low-code) |
| **Solution Package** | `MSKK_InquirySolution_x_x_x_x.zip` |

### Deployment Phases

| Phase | Name | Description | Est. Time |
|-------|------|-------------|-----------|
| **Phase 1** | Environment & Infrastructure Setup | Provision SharePoint lists, document library, and verify Power Platform environment | 60–90 min |
| **Phase 2** | Solution Deployment & Configuration | Import solution package, load sample data, configure environment variables, flows, and security roles | 60–90 min |
| **Phase 3** | Agent Build & Publishing | Re-link knowledge sources, verify agent instructions and actions, test, publish, and deploy to M365/Teams | 60–90 min |
| **Advanced** | Azure Extension *(Optional)* | Provision Azure AI Search + MCP Server for extended knowledge retrieval | 2–4 hours |

> 💡 **Tip:** Complete each phase fully and verify the exit criteria before proceeding to the next phase.

## Phase 1 — Environment & Infrastructure Setup

**Goal:** Create the SharePoint FAQ list and document library that will serve as the agent's knowledge sources and data storage locations.

---

### Step 1-1 — Create SharePoint FAQ List

The FAQ list is the primary knowledge source for the agent. It stores structured question-and-answer data that the agent will retrieve using generative AI.

**1.** Navigate to your SharePoint site (the site you will use for this agent deployment).

**2.** In the top menu bar, click **New** → select **List**.

**3.** When prompted for the list creation method, select **From CSV**.

**4.** Click **Upload file** and select the **`FAQ.csv`** file from the solution package.

**5.** Review the CSV content preview and click **Next**.

**6.** Keep the list name as **`FAQ`** and click **Create**.

**7.** Verify that the FAQ list appears in the site navigation on the left-hand side.

**8.** Click **All items** in the view selector → click **Edit columns** (or "Add or remove fields") → **uncheck the `Title` column** from the view.

**9.** Adjust the column widths of the **Question** and **Answer** columns for readability.

**10.** Configure the Title column as non-required:
   - Click the gear icon ⚙️ (top right) → **List settings**
   - Under **Columns**, click **Title**
   - Set **"Require that this column contains information"** to **No**
   - Click **OK** to save

> ✅ **Verification:** The FAQ list should display items with "Question" and "Answer" columns visible. The "Title" column should no longer be required.

> 💡 **Important:** **Record the FAQ List URL** — you will need this URL when configuring the knowledge source in Copilot Studio (Step 3-1). Copy the URL from your browser's address bar while viewing the FAQ list.

---

### Step 1-2 — Create SharePoint Document Library for Application Templates

The document library stores application form templates and is also the target location for completed Excel reports generated by Power Automate.

**1.** In the same SharePoint site, click **New** → **Document library**.

**2.** Name the library **`ApplicationTemplates`** (or a name consistent with your organization's naming convention).

**3.** Click **Create**.

**4.** Upload the **Excel report template file** (included in the solution package) to this library:
   - Click **Upload** → **Files**
   - Select the Excel template file from the solution package
   - Click **Open**

**5.** Create a subfolder named **`Reports`** within the library to store completed application reports:
   - Click **New** → **Folder**
   - Name it **`Reports`** → click **Create**

> ✅ **Verification:** The document library should contain the Excel template file and a `Reports` subfolder.

> 💡 **Important:** **Record the Document Library URL** — you will need this when configuring environment variables in Step 2-5.

---

### Step 1-3 — Verify Power Platform Environment

**1.** Navigate to [Power Platform Admin Center](https://admin.powerplatform.microsoft.com/).

**2.** Confirm that a **dedicated (non-default) Power Platform environment** exists and is in an **active** state.

**3.** Verify that **Dataverse** is enabled for the target environment.

**4.** Confirm that your account has the **System Administrator** security role in the target environment.

**5.** Confirm that **Copilot Studio** is accessible within the target environment by navigating to [https://copilotstudio.microsoft.com/](https://copilotstudio.microsoft.com/) and selecting the environment.

> ✅ **Phase 1 Exit Criteria:**
> - [ ] FAQ List created in SharePoint with data imported from `FAQ.csv`
> - [ ] Title column set to non-required in the FAQ list
> - [ ] FAQ List URL recorded
> - [ ] Document Library created with Excel template uploaded and `Reports` subfolder created
> - [ ] Document Library URL recorded
> - [ ] Power Platform environment verified with Dataverse enabled

## Phase 2 — Solution Deployment & Configuration

**Goal:** Import the solution package into Power Platform, load sample data into Dataverse, configure environment variables, activate flows, and assign security roles.

---

### Step 2-1 — Import Solution Package

**1.** Navigate to [https://copilotstudio.microsoft.com/](https://copilotstudio.microsoft.com/) and select the **target environment**.

**2.** In the left navigation, click **⋯ (More)** → click **Solutions**.

**3.** Click **Import solution**.

**4.** Click **Browse** and select the solution file: **`MSKK_InquirySolution_x_x_x_x.zip`**.

**5.** Click **Next**.

**6.** Click **Next** again on the solution overview screen.

**7.** On the **Connections** step:
   - All required connectors must show a green **✓ checkmark** before you can proceed
   - If any connector shows an error or is not connected:
     - Click the connector → sign in / re-authenticate
     - Wait for the checkmark to appear
     - Repeat for all connectors

> ⚠️ **Warning:** Do **not** click "Import" until **all connectors show a green checkmark**. Importing with disconnected connectors will result in broken flows that require manual repair.

**8.** On the **Environment Variables** step, configure the following:

| Variable | Value |
|----------|-------|
| SharePoint Site URL | The URL of your SharePoint site (from Step 1-1) |
| FAQ List Name | `FAQ` (or the name you used in Step 1-1) |
| Document Library Name | `ApplicationTemplates` (or the name from Step 1-2) |
| Report Template File Name | The exact file name of the Excel template uploaded in Step 1-2 |

**9.** Click **Import** and wait for the import to complete (typically 2–5 minutes).

**10.** Verify the **MSKK Inquiry Solution** appears in the solutions list.

**11.** Click **Publish all customizations** to ensure all components are fully activated.

> ✅ **Verification:** The solution should appear in the solutions list with status "Succeeded". The agent `WorkplaceAgent` should be visible in Copilot Studio.

---

### Step 2-2 — Import Category Data (Dataverse)

**1.** Navigate to [https://make.powerapps.com/](https://make.powerapps.com/) and select the **target environment**.

**2.** In the left navigation, click **Tables**.

**3.** Search for and select the **Category Data** table.

**4.** Click **Import** → **Import data from Excel**.

**5.** Click **Upload** and select the **`CategoryData.csv`** file from the solution package.

**6.** Verify the **Mapping status** shows **"Mapping completed successfully"**.
   - If mapping shows errors, check that the column names in the CSV match the Dataverse table column names exactly.

**7.** Click **Import**.

**8.** When the success message **"Import completed successfully"** appears, close the dialog and refresh the browser to confirm data is visible in the table.

---

### Step 2-3 — Import Inquiry Ticket Sample Data (Dataverse)

**1.** In the Power Apps maker portal, navigate to **Tables**.

**2.** Search for and select the **Inquiry Ticket** table (`mskk_inquiryticket`).

**3.** Click **Import** → **Import data from Excel**.

**4.** Click **Upload** and select the **`InquiryData.csv`** file from the solution package.

**5.** Verify the **Mapping status** shows **"Mapping completed successfully"**.

**6.** Click **Import**.

**7.** When the success message appears, close the dialog and **refresh the browser** to confirm sample inquiry records are visible.

> 💡 **Tip:** The sample inquiry data provides realistic test records for verifying the agent's inquiry history retrieval functionality.

---

### Step 2-4 — Import Internal FAQ Data (Dataverse)

**1.** Navigate to **Tables** → search for and select the **Internal FAQ** table (`mskk_FAQ`).

**2.** Click **Import** → **Import data from Excel**.

**3.** Click **Upload** and select the **`FAQData.csv`** file from the solution package.

**4.** Verify the mapping status and click **Import**.

**5.** Confirm successful import and refresh the browser.

---

### Step 2-5 — Configure Environment Variables

**1.** In the Power Apps maker portal, open the **MSKK Inquiry Solution**.

**2.** Navigate to **Environment variables** within the solution.

**3.** Review and update each variable if it was not set during the solution import (Step 2-1):

| Variable | Expected Value |
|----------|----------------|
| SharePoint Site URL | Full URL of your SharePoint site |
| FAQ List Name | `FAQ` |
| Document Library Name | `ApplicationTemplates` |
| Report Template File Name | File name of the Excel report template (e.g., `ReportTemplate.xlsx`) |
| Reports Folder Name | `Reports` |
| Approval Notification Target | Teams channel ID or manager email for approval routing |

**4.** Click **Save** for each variable you update.

---

### Step 2-6 — Activate and Verify Power Automate Flows

**1.** In the solution, navigate to **Cloud flows**.

**2.** Verify all flows are in **On** (active) status. If any flow shows as **Off**, click the flow → click **Turn on**.

**3.** Flows to verify:

| Flow Name | Trigger | Status |
|-----------|---------|--------|
| Create Inquiry Ticket | Called by agent when no FAQ match | ☐ On |
| Get Inquiry History | Called by agent on user request | ☐ On |
| Submit Application Request | Called by agent when application confirmed | ☐ On |
| Approval Flow | Triggered by Submit Application Request | ☐ On |
| Excel Write-back Flow | Triggered after approval decision | ☐ On |
| FAQ Recording Flow | Triggered when ticket is resolved | ☐ On |

**4.** Perform a **manual test run** for each flow using the **Test** button in Power Automate to verify connections are working.

> ⚠️ **Warning:** If a flow shows connection errors during testing, navigate to the flow's **Edit** view → check each connector's connection reference → re-authenticate if needed.

---

### Step 2-7 — Configure Security Roles

**1.** Navigate to [Power Platform Admin Center](https://admin.powerplatform.microsoft.com/).

**2.** Select the target environment → click **Users**.

**3.** Select a user to assign a role → click **Manage security roles**.

**4.** Assign the appropriate security role:

| User Type | Security Role to Assign |
|-----------|------------------------|
| End users (employees using the agent) | **Inquiry Agent User** |
| Support staff / administrators | **Inquiry Administrator** |

**5.** Repeat for all users who will access the agent or manage inquiries.

> ✅ **Phase 2 Exit Criteria:**
> - [ ] Solution package imported successfully with all connectors connected
> - [ ] "Publish all customizations" completed
> - [ ] Category data, inquiry sample data, and internal FAQ data imported into Dataverse
> - [ ] All environment variables configured
> - [ ] All 6 Power Automate flows are active (On)
> - [ ] Security roles assigned to relevant users

## Phase 3 — Agent Build & Publishing

**Goal:** Re-link knowledge sources in Copilot Studio, verify agent instructions and actions, validate end-to-end functionality, publish, and deploy to Microsoft 365 Copilot and Teams.

---

### Step 3-1 — Re-link Knowledge Sources in Copilot Studio

After solution import, the SharePoint knowledge source may appear as disconnected. This step restores the knowledge connection.

**1.** Navigate to [https://copilotstudio.microsoft.com/](https://copilotstudio.microsoft.com/) and select the target environment.

**2.** Open the **WorkplaceAgent**.

**3.** Navigate to the **Knowledge** section in the left panel.

**4.** If the SharePoint FAQ knowledge source shows as disconnected or missing:
   - Click **Add knowledge** → select **SharePoint**
   - Paste the **FAQ List URL** recorded in Step 1-1
   - Click **Add**

**5.** Verify the Document Library is also linked as a knowledge source:
   - If missing, click **Add knowledge** → **SharePoint**
   - Enter the **Document Library URL** recorded in Step 1-2
   - Click **Add**

**6.** ⚠️ **Critical:** Click the **Web search** toggle and ensure it is **disabled (Off)**.
   - This ensures the agent only returns answers grounded in your organizational knowledge sources and does not use general LLM knowledge.

> ✅ **Verification:** The Knowledge section should show two active SharePoint knowledge sources (FAQ List and Document Library) with web search disabled.

---

### Step 3-2 — Verify Agent Instructions (System Prompt)

**1.** In the **WorkplaceAgent**, navigate to the **Instructions** (system prompt) section.

**2.** Review the system prompt to confirm it covers:
   - Agent identity: internal inquiry and application request handling assistant
   - FAQ behavior: search knowledge base and return grounded answers with source reference
   - Ticket creation behavior: create a support ticket when no FAQ answer is found
   - Application handling: guide the user through required form fields step-by-step
   - Fallback: inform the user when the topic is out of scope
   - Tone: professional and helpful; respond in the same language as the user

**3.** Customize the system prompt if needed for your organization (e.g., update the company name, out-of-scope message, or specific application types supported).

---

### Step 3-3 — Verify Actions / Tools

**1.** Navigate to **Actions** (or **Tools**) in the **WorkplaceAgent** configuration.

**2.** Confirm all 5 actions are present and connected:

| Action Name | Connected Flow | Status |
|-------------|---------------|--------|
| Create Inquiry Ticket | Create Inquiry Ticket Flow | ☐ Active |
| Get Inquiry History | Get Inquiry History Flow | ☐ Active |
| Submit Application Request | Submit Application Request Flow | ☐ Active |
| Send Approval Notification | Approval Flow | ☐ Active |
| Write to Excel Report | Excel Write-back Flow | ☐ Active |

**3.** If any action appears disconnected, re-link it to the corresponding Power Automate flow.

---

### Step 3-4 — Test Agent in Copilot Studio Test Chat

**1.** Click **Test your agent** (top right) to open the test chat panel.

**2.** Run the following test cases and verify expected behavior:

**Scenario A — FAQ Inquiry:**

| Test Input | Expected Behavior |
|------------|-------------------|
| `"What is the company Wi-Fi SSID and password?"` | Agent searches FAQ and returns a grounded answer with source reference |
| `"Is a side job allowed?"` | Agent returns the FAQ answer on the side job policy |
| A question that does NOT exist in the FAQ | Agent responds that it could not find an answer and creates a support ticket; provides the ticket number |

**Scenario B — Application Request:**

| Test Input | Expected Behavior |
|------------|-------------------|
| `"I would like to borrow office equipment"` | Agent guides the user through the application form fields step-by-step |
| Complete the application form fields | Agent confirms submission details and asks for confirmation |
| Confirm submission | Agent confirms the application has been submitted and the approval process has started |

> ✅ **Verification:** All test scenarios produce the expected outputs. Ticket creation and application flow triggering can be confirmed in Dataverse and Power Automate run history.

---

### Step 3-5 — Publish Agent

**1.** In Copilot Studio, click **Publish** in the top navigation bar.

**2.** Click **Publish** in the confirmation dialog.

**3.** Verify the **last published date** shows today's date.

> 💡 **Tip:** Any changes made to the agent (instructions, knowledge, actions) require republishing before they take effect in production.

---

### Step 3-6 — Deploy to Microsoft 365 Copilot & Teams

**1.** After publishing, click **Channels** in the left navigation.

**2.** Under **Microsoft channels**, click **Teams and Microsoft 365 Copilot**.

**3.** Verify **"Make agent available in Microsoft 365 Copilot"** is **checked**.

**4.** Click **Add channel**.

**5.** Click **"View agent in Microsoft 365"** — this opens the Microsoft 365 Copilot interface.

**6.** In the M365 Copilot interface, click **Add** to add the agent to your Copilot experience.

**7.** Test in **Microsoft Teams**:
   - Open Teams → search for the agent under **Built for your org** in the app catalog
   - Start a conversation with the agent
   - Send a test inquiry and verify a response is returned

> ✅ **Verification:** The agent appears under "Built for your org" in M365 Copilot and/or Teams and responds correctly to test messages.

---

### Step 3-7 — Functional Verification (End-to-End Test)

Run the full end-to-end validation using the evaluation test cases.

**1.** Open the evaluation CSV file (`EvaluationTestCases.csv`) provided in the solution package.

**2.** For each test case:
   - Send the question to the agent via M365 Copilot or Teams
   - Compare the actual response to the expected response in the CSV
   - Mark pass/fail based on the passing score threshold (TextSimilarity ≥ 70–80%)

**3.** Run the **application request end-to-end test**:
   - Submit a test application through the agent
   - Verify the approval notification is received by the approver (Teams adaptive card or Outlook email)
   - Approve the request and confirm:
     - Application status updated in Dataverse
     - Result notification sent to the requestor
     - Excel report file generated and saved to the SharePoint Document Library `Reports` folder

> ✅ **Phase 3 Exit Criteria:**
> - [ ] Both SharePoint knowledge sources linked and web search disabled
> - [ ] Agent instructions reviewed and customized
> - [ ] All 5 actions verified as active and connected
> - [ ] Test chat passed for both Scenario A (FAQ) and Scenario B (Application)
> - [ ] Agent published successfully
> - [ ] Teams and M365 Copilot channel added
> - [ ] End-to-end functional verification completed
> - [ ] Evaluation test case pass rate meets the defined threshold

## Troubleshooting Guide

| # | Symptom | Likely Cause | Resolution |
|---|---------|-------------|------------|
| 1 | FAQ knowledge source shown as disconnected after import | Knowledge source connections are not preserved during solution import | Re-add the SharePoint knowledge source manually in Copilot Studio (Step 3-1) |
| 2 | Connector shows error during solution import | Authentication token expired or credentials need refresh | Click the connector → sign in / re-authenticate → verify the green checkmark appears before importing |
| 3 | Dataverse import fails with mapping error | Column names in CSV do not match Dataverse table schema | Verify column names in your CSV file exactly match the Dataverse table column names; re-upload the corrected file |
| 4 | Power Automate flow is in "Off" state after import | Flows are imported in a disabled state and must be manually turned on | Navigate to the flow → click **Turn on** → verify the flow is active |
| 5 | Approval flow not triggering | Flow is inactive or connection references are not set | Verify flow is On; check all connection references in the solution are configured with valid accounts |
| 6 | Agent returns answers from general knowledge (not FAQ) | Web search is enabled in the agent's knowledge settings | Disable the **Web search** toggle in the Knowledge section of the agent |
| 7 | Agent not visible in Teams or M365 Copilot | Channel not added or agent not yet published | Ensure the agent is published (Step 3-5) and the Teams/M365 Copilot channel is added (Step 3-6) |
| 8 | Excel report not generated after approval | Report template file not found or file name mismatch | Verify the Excel template exists in the SharePoint Document Library and the file name exactly matches the environment variable value |
| 9 | Security role error when accessing the agent | User does not have the required Dataverse security role | Assign the **Inquiry Agent User** role via Power Platform Admin Center (Step 2-7) |
| 10 | Agent creates a ticket even when FAQ has the answer | Agent confidence threshold too low or FAQ content insufficient | Review FAQ list content for completeness; adjust agent instructions to tune FAQ matching behavior |

## Advanced Extension Setup (Optional) — Azure AI Search + MCP Server

This section describes how to extend the agent with Azure AI Search and an MCP Server (hosted on Azure Functions) for hybrid and vector search over extended knowledge sources.

> ⚠️ **Note:** This extension requires an **Azure subscription** and additional permissions. Complete the basic deployment (Phases 1–3) before starting this extension.

---

### Step A-1 — Provision Azure Resources

**1.** In Azure Portal, create a new **Resource Group** (e.g., `rg-workplace-agent`).

**2.** Provision the following Azure resources within the resource group:

| Resource | SKU / Plan | Purpose |
|----------|-----------|---------| 
| Azure AI Search | Basic (minimum) | Hybrid and vector search over extended knowledge |
| Azure Blob Storage | Standard LRS | Extended document corpus storage |
| Azure Functions | Consumption or Premium plan | Hosts the MCP Server |
| Azure AI Document Intelligence *(optional)* | S0 | PDF text extraction for document ingestion |

**3.** Register the required resource providers in your Azure subscription if not already registered:
   - `Microsoft.Search`
   - `Microsoft.CognitiveServices`
   - `Microsoft.Storage`
   - `Microsoft.Web`

> 💡 **Tip:** Use infrastructure-as-code templates (`main.bicep`) included in the **Azure Extension** folder of the solution package to automate resource provisioning.

---

### Step A-2 — Deploy MCP Server to Azure Functions

**1.** Download the MCP Server code package from the **`Azure Extension`** folder in the solution source.

**2.** Deploy the Azure Function App using one of the following methods:
   - **Azure CLI:** `func azure functionapp publish <your-function-app-name>`
   - **VS Code:** Use the Azure Functions extension → Deploy to Function App

**3.** Configure the following **Application Settings** in the Azure Function App:

| Setting Name | Value |
|-------------|-------|
| `AZURE_SEARCH_ENDPOINT` | `https://<your-search-service>.search.windows.net` |
| `AZURE_SEARCH_FAQ_INDEX` | `faq-index` |
| `AZURE_SEARCH_INQUIRY_INDEX` | `inquiry-index` |
| `AZURE_SEARCH_TICKET_INDEX` | `ticket-index` |
| `AZURE_STORAGE_ACCOUNT_NAME` | Your storage account name |
| `AZURE_STORAGE_CONTAINER_NAME` | `data` |
| `FUNCTIONS_WORKER_RUNTIME` | `python` |

**4.** Note the **Function App URL** and **Function Key** — you will need these in Step A-4.

---

### Step A-3 — Configure Azure AI Search Indexes

**1.** Upload extended knowledge documents (manuals, policy PDFs, historical inquiry data) to the Azure Blob Storage container.

**2.** In Azure Portal, navigate to your **Azure AI Search** service.

**3.** Create the following indexes:

| Index Name | Content |
|------------|---------|
| `faq-index` | FAQ knowledge base content |
| `inquiry-index` | Historical inquiry ticket data |
| `ticket-index` | Active inquiry ticket records |

**4.** Configure **indexers** to pull data from Azure Blob Storage and run them to populate the indexes.

**5.** Verify indexed document count in Azure Portal → Search service → **Indexes**.

> 💡 **Tip:** Enable **Semantic Search** (free tier available) on your Azure AI Search instance for improved answer relevance.

---

### Step A-4 — Connect MCP Server to Copilot Studio Agent

**1.** In Copilot Studio, open the **WorkplaceAgent**.

**2.** Navigate to **Tools** (or **Actions**) → click **Add a tool**.

**3.** Select **Model Context Protocol (MCP)**.

**4.** Enter the **MCP Server URL** (your Azure Functions endpoint URL).

**5.** Configure authentication using the **Function Key** from Step A-2.

**6.** Verify that the three MCP tools are discovered:
   - FAQ Search Tool
   - Inquiry History Search Tool
   - Ticket Creation API Tool

**7.** Test each MCP tool in the agent test chat:
   - Ask a question that requires searching beyond the SharePoint FAQ
   - Verify the agent uses the MCP tool to retrieve results from Azure AI Search

---

### Step A-5 — Enable Power Platform VNet Support *(if required)*

Required only if your organization mandates private connectivity between Power Platform and Azure services.

**1.** Follow the **`PowerPlatform-VNetSupport-Guide.pdf`** included in the Azure Extension folder.

**2.** Register the required provider in your Azure subscription: `Microsoft.PowerPlatform`.

**3.** Create a VNet and subnet, configure NSG, and delegate the subnet to `Microsoft.PowerPlatform/enterprisePolicies`.

**4.** Create an **Enterprise Policy** using the PowerShell scripts provided in the solution package.

**5.** In **Power Platform Admin Center**, associate the target environment with the Enterprise Policy.

**6.** Verify private connectivity by testing a flow that connects to Azure AI Search through the VNet.
