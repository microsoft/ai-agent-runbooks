# Step-by-Step Guide — Workplace Agent

**Scenario**: Internal / External Inquiry Routing and Microsoft 365 Copilot License Request Automation  
**Platform**: Microsoft Copilot Studio + Power Platform (SharePoint, Dataverse, Power Automate, Power Apps, Teams)  
**Target Readers**: Solution Architect / Delivery Engineer / Power Platform Administrator / Copilot Studio Maker  
**Estimated Effort**: Intermediate setup with standard Power Platform administration tasks

---

## 1. Prerequisites

Before you begin, confirm that the following are available.

### Permissions & Materials

| # | Prerequisite |
|---|--------------|
| 1 | Installation package: `WorkplaceAgent_x_x_x_x.zip` or the latest provided unmanaged solution package. |
| 2 | Target Power Platform environment with Dataverse enabled and language set to English. |
| 3 | System Administrator permissions in the target environment. |
| 4 | Edit permission on the SharePoint site where the FAQ list will be created. |
| 5 | `FAQ_en.csv`, used to create the SharePoint Public FAQ list. |
| 6 | Permission to configure Copilot Studio agent knowledge, tools, Web search, and Teams publishing. |

### Licensing

| Item | Requirement |
|------|-------------|
| Copilot Studio | Copilot Studio license, or Microsoft 365 Copilot entitlement that includes Copilot Studio capacity. |
| Power Apps Premium | Licensing that covers Dataverse and any premium connectors used by the solution. |
| Dataverse | Sufficient Dataverse database capacity for inquiry and license request tables. |
| Microsoft 365 Copilot | Required if the agent will be surfaced through Microsoft 365 Copilot. |

### SharePoint FAQ List Schema

Create the FAQ list from `FAQ_en.csv`. The expected columns are:

| Display name | Internal name | Type | Required | Notes |
|--------------|---------------|------|----------|-------|
| Title | `Title` | Single line of text | No | Default SharePoint column. |
| Question | `mskk_question` | Multiple lines of text | No | FAQ question text. |
| Answer | `mskk_answer` | Multiple lines of text | No | FAQ answer text. |
| Category | `mskk_category` | Choice | Yes | HR / Legal / IT / Other. |

## 3. Step 1 — Create the SharePoint Public FAQ List

1. Open the target SharePoint site where the FAQ should live.
2. Select **New** → **List**.
3. Choose **From Excel** or **From CSV**.
4. Upload `FAQ_en.csv`.
5. Confirm that the import preview recognizes these columns:
   - `Question`
   - `Answer`
   - `Category`
6. Name the list, for example: **Company FAQ**.
7. Create the list.
8. Set the default **Title** column to **not required**:
   - Open list settings.
   - Select **Title**.
   - Set **Require that this column contains information** to **No**.
   - Save.
9. Record the following values for solution import:
   - SharePoint site URL, for example `https://contoso.sharepoint.com/sites/workplace`.
   - FAQ list name, for example `Company FAQ`.

> 💡 The FAQ import file contains sample HR, Legal, IT, and Other entries. Replace or extend these entries with organization-approved content before production rollout.  
> 💡 Access permissions to the FAQ list must be granted to Workplace Agent users.

---

## 4. Step 2 — Import the Solution

The package is an **unmanaged solution**. Importing it adds the Copilot Studio agent, model-driven app, Dataverse components, security roles, and cloud flows to the target environment.

1. Open **Power Apps**: `https://make.powerapps.com`.
2. Select the target environment.
3. Go to **Solutions**.
4. Select **Import solution**.
5. Browse to `WorkplaceAgent_x_x_x_x.zip`.
6. Select **Next**.
7. On the connections screen, create or select the required connections:
   - Microsoft Dataverse
   - SharePoint
   - Office 365 Users, if prompted by the imported flows
   - Microsoft Teams
8. On the environment variables screen, enter the SharePoint values from Step 1.

| Environment variable | Value |
|----------------------|-------|
| SharePoint Site URL | The SharePoint site URL that hosts the FAQ list. |
| FAQ List | The SharePoint list name created from `FAQ_en.csv`. |

9. Select **Import** and wait until the import completes.
10. Confirm that the **WorkplaceAgent** solution appears in the solution list.

> ⚠️ If import fails, confirm that the target environment has Dataverse enabled and that the importing user has System Administrator permissions.

---

## 5. Step 3 — Validate Environment Variables

1. In **Power Apps**, open **Solutions** → **WorkplaceAgent**.
2. Open **Environment variables** and confirm:
   - SharePoint Site URL is correct.
   - FAQ list name matches the created SharePoint list.
3. If any value is missing or incorrect, edit the environment variable and save.

---

## 6. Step 4 — Turn On and Validate Cloud Flows

Cloud flows may be off immediately after solution import.

1. In the **WorkplaceAgent** solution, open **Cloud flows**.
2. Validate each flow:

| Flow | Purpose | Validation |
|------|---------|------------|
| **Ticket creation flow** | Creates inquiry tickets in Dataverse and notifies requesters. | Confirm it is on and has Dataverse / Teams connections. |
| **Answer Delivery Flow** | Sends administrator answers to requesters through Teams. | Confirm it is on and can access Dataverse and Teams. |
| **FAQ publishing flow** | Publishes final ticket answers to the SharePoint Public FAQ. | Confirm it is on and can access Dataverse and SharePoint. |
| **License Request Flow** | Registers Microsoft 365 Copilot license requests in Dataverse. | Confirm it is on and can access Dataverse. |

3. Open any flow with an error and fix the connection.
4. Save and turn on each flow.

---

## 7. Step 5 — Assign Security Roles

The solution includes two custom Dataverse security roles.

1. Open the **Power Platform admin center**: `https://admin.powerplatform.microsoft.com`.
2. Go to **Environments** and select the target environment.
3. Open **Settings** → **Users + permissions** → **Users**. 
4. Select a user.
5. Choose **Manage security roles**.
6. Assign the appropriate role:

| Role | Assign To | Purpose |
|------|-----------|---------|
| **Inquiry administrator** | Support, HR, IT, Legal, or back-office staff | Manage tickets and license requests in the Workplace Management App. |
| **Inquirer** | Employees who will use the agent | Ask questions, create tickets, and submit license requests. |

7. Save role assignments.

> 💡 Users need both the correct security role and app sharing permissions to use the model-driven app.  
  💡 Reference: [Assign security roles - Power Platform | Microsoft Learn](https://learn.microsoft.com/en-us/power-platform/admin/assign-security-roles)
---

## 8. Step 6 — Publish and Share the Workplace Management App

1. In **Power Apps**, open **Solutions** → **WorkplaceAgent**.
2. Open **Apps**.
3. Locate **Workplace Management App**.
4. Select **Publish**, if a publish action is available.
5. Select **Share**.
6. Add the inquiry administrator users or an Entra ID security group.
7. Assign the **Inquiry administrator** role when sharing.
8. Select **Play** and confirm the app opens.
9. Verify that administrators can see:
   - Inquiry Ticket table.
   - M365 Copilot License Request table.

---

## 9. Step 7 — Configure the Copilot Studio Agent

1. Open **Copilot Studio**: `https://copilotstudio.microsoft.com`.
2. Select the same target environment.
3. Open the imported **Workplace Agent**.

### 9.1 Configure Knowledge — SharePoint Public FAQ

1. Open the agent **Knowledge** section.
2. Remove any default or placeholder FAQ source that is not the target SharePoint list.
3. Select **Add knowledge**.
4. Choose **SharePoint**.
5. Add the SharePoint FAQ list created in Step 1.
6. Save the knowledge configuration.


---

## 10. Step 8 — Publish the Agent

1. In Copilot Studio, select **Publish**.
2. Wait for publishing to complete.
3. Open **Channels**.
4. Select **Microsoft Teams**.
5. Add or update the Teams channel configuration.
6. If tenant governance requires approval, submit the agent for admin approval.
7. Confirm that the agent is available to target users.

---

## 11. Step 9 — Verify the Scenario

### 11.1 First-run Connection Consent

On first use, users may see a connection message such as:

> "Let's get you connected first. Open connection manager to verify your credentials."

If this appears:

1. Select **Open connection manager**.
2. Authorize the required connections.
3. Return to the conversation.
4. Retry the prompt.

### 11.2 Internal FAQ Answer Test

Prompt:

```text
I'd like to learn about the childcare leave and family care leave programs.
```

Expected result:

- The agent classifies the inquiry as internal.
- The agent answers from the SharePoint Public FAQ.
- The agent asks whether the issue was resolved.

### 11.3 Internal Unresolved Ticket Test

Prompt:

```text
I have an internal workplace question that is not in the FAQ. Can you help me create a ticket?
```

Expected result:

- The agent starts ticket creation.
- A Dataverse Inquiry Ticket record is created.
- The inquiry body and AI-generated answer are stored.
- The requester receives a Teams notification.

### 11.4 External Microsoft-product Test

Prompt:

```text
What can Copilot in Excel do?
```

Expected result:

- The agent classifies the question as external.
- The agent identifies it as a Microsoft-product question.
- The agent uses Microsoft Learn Docs MCP.

### 11.5 External Non-Microsoft Test

Prompt:

```text
What are common best practices for organizing a personal task list?
```

Expected result:

- The agent classifies the question as external and non-Microsoft.
- The agent uses Web search.

### 11.6 License Request Test

Prompt:

```text
I would like to request a Microsoft 365 Copilot license.
```

Expected result:

- The license request topic starts.
- The agent collects required request information.
- The License Request Flow creates a Dataverse record.

### 11.7 Administrator App Test

1. Open **Workplace Management App**.
2. Confirm that the newly created inquiry ticket appears.
3. Enter an administrator answer.
4. Run the **Send answer** action.
5. Confirm that the requester receives a Teams notification.
6. Run the **Publish** action.
7. Confirm that the answer is copied to the SharePoint Public FAQ.

## 13. Troubleshooting

| Symptom | What to Check |
|---------|---------------|
| Agent shows first-run connection consent | Open connection manager, authorize required connections, then retry. |
| Internal FAQ answer is not returned | Confirm the SharePoint FAQ list is added as knowledge and the agent was republished. |
| Agent uses external sources for internal questions | Review the agent instructions and routing topic logic. |
| Ticket is not created | Confirm Ticket creation flow is on, connections are valid, and Dataverse permissions are assigned. |
| Teams notification is not sent | Check Teams connector connection, flow run history, and requester identity. |
| License request is not registered | Confirm License Request Flow is on and the Dataverse connection is valid. |
| FAQ publishing fails | Confirm SharePoint environment variables, list name, and flow connection permissions. |
| User cannot use the agent | Confirm the user has access to the published channel and the Inquirer role. |
| Administrator cannot open Workplace Management App | Confirm app sharing and Inquiry administrator role assignment. |
| Microsoft-product answers are unavailable | Confirm Microsoft Learn Docs MCP is configured and enabled. |
| Web answers are unavailable | Confirm Web search is enabled and allowed by tenant policy. |

---

## 14. Uninstall / Cleanup

1. In **Power Apps** → **Solutions**, select the **WorkplaceAgent** solution.
2. Delete the solution.
3. Delete the SharePoint FAQ list if it is no longer needed.
4. Remove security role assignments from users.
5. Remove or disable related connections if they are not used elsewhere.
