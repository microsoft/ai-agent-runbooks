# Enterprise RAG (Multi-Source, Governed) Pattern

> **Most widely deployed pattern.**
> Grounds every LLM response in real organizational data
> with document-level access control and full citation trails.

---

## 🏆 Why This Pattern Matters

Enterprise users need answers that are **accurate, auditable, and access-controlled**.
Generic LLMs hallucinate. This pattern eliminates that risk by grounding every response
in your actual organizational documents — with a traceable citation back to the source.

| Signal | Evidence |
|--------|----------|
| **Industries** | Insurance, Healthcare, Public Sector, Finance, Legal, HR, IT, Sales |
| **Data sources** | SharePoint, OneDrive, Dataverse, Azure SQL, Microsoft Fabric, Dynamics CRM |
| **Task types** | Q&A, summarization, policy interpretation, compliance checks, knowledge access |

---

## 💡 Core Value Proposition

> **Enterprise RAG turns your document repositories into a governed, AI-searchable knowledge base —
> where every answer is grounded in real data, access-controlled per user, and traceable to its source.**

---

## ⏰ When to Use / When to Avoid

| ✅ Use when... | ❌ Avoid when... |
|----------------|-----------------|
| Large document repositories (500+ files) | Knowledge base fits in < 50 short documents |
| Access-controlled content is required | All users have identical access to all data |
| Auditability and citations are required | Speed is critical and citations aren't needed |
| Multiple data sources need unified search | Data lives in a single, well-structured database |
| Hallucination risk must be minimized | Real-time computation is the primary need |
